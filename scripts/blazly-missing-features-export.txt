Blazly SEO integrations
Integrations
Located in the left navigation menu under the Workspace section.
Integrations allow users to connect external platforms, content management systems, analytics tools, and publishing endpoints directly with Blazly. These integrations enable content publishing, performance analysis, website management, and workflow automation from a single platform.
Connected integrations can be used across content generation, SEO analysis, publishing workflows, and automated content distribution.
Features
Connected Integrations Counter
Displays the total number of active integrations currently connected to the account.
This provides a quick overview of all connected publishing and analytics platforms.
Google Search Console
Google Search Console integration provides access to website performance and search visibility data directly within Blazly.
This integration is required for:
* SEO Audit
* Page Speed Analysis
* Keyword Performance Analysis
* Search Visibility Monitoring
* Search Analytics Reporting
Connect GSC
Allows users to securely connect their Google Search Console account through OAuth authentication.
Once connected, Blazly can access verified website properties and retrieve performance data for analysis tools.
Search Performance Data
After connection, Blazly can access:
* Search impressions
* Clicks
* Click-through rate (CTR)
* Average ranking positions
* Search queries
* Website performance insights
CMS Integrations
CMS Integrations allow users to connect website platforms directly with Blazly for content publishing and management.
Webflow
Allows users to connect Webflow websites to Blazly.
Connected Webflow sites can be used for:
* Content publishing
* Blog deployment
* Website content management
* Automated publishing workflows
Add Site
Connects a Webflow website to Blazly and makes it available for publishing operations.
WordPress
Allows users to connect WordPress websites directly with Blazly.
Connected WordPress sites can be used to publish generated content without requiring manual content transfer.
Add Site
Connects a WordPress website and authorizes Blazly to manage content publishing activities.
GoHighLevel
Allows users to connect GoHighLevel locations and websites through OAuth authentication.
Connected GoHighLevel accounts can receive content generated within Blazly and support direct publishing workflows.
Connect
Initiates the OAuth connection process and authorizes Blazly to access the selected GoHighLevel account.
Direct Blog Publishing
Once connected, users can publish blog content directly to supported GoHighLevel websites without leaving Blazly.
Webhook Integration
Webhook Integration enables content publishing to virtually any platform that supports HTTPS webhooks.
This integration is designed for advanced publishing workflows and third-party automation platforms such as:
* Zapier
* Make
* n8n
* Custom APIs
* Internal publishing systems
* Proprietary CMS platforms
HTTPS Webhook URL
Allows users to configure a destination webhook endpoint where content payloads will be delivered.
The webhook receives structured JSON data whenever content is published through Blazly.
Fixed Blazly Bridge URL
Provides a persistent inbound endpoint generated by Blazly.
This bridge URL can be used when external systems require a stable endpoint while Blazly forwards requests to the configured webhook destination.
Override URL for Test Only
Allows users to temporarily specify an alternative webhook endpoint for testing purposes without modifying the saved production webhook configuration.
Save Webhook
Stores the webhook configuration for future publishing operations.
Saved webhook settings can be reused across multiple content publishing workflows.
Send Test JSON
Sends a sample payload to the configured webhook endpoint.
This allows users to validate:
* Endpoint connectivity
* Payload structure
* Automation workflows
* Third-party integrations
before using live publishing.
Webhook Payload
The test payload contains sample content data structured similarly to live publishing events.
Payloads may include:
* URL
* Title
* Meta Title
* Meta Description
* Content (HTML)
* Primary Keyword
* Publishing Metadata
Send to Webhook
Once configured, users can publish content directly to the connected webhook endpoint from supported areas of the platform, including:
* My Blogs
* Bulk Publishing Workflows
* Automated Content Distribution Pipelines
This enables fully automated publishing workflows across external platforms and services.

Blazly GEO projects page
Prompt Ranking
The Prompt Ranking feature helps users discover the prompts and questions that Large Language Models (LLMs) are most likely to use when recommending their brand. It analyzes the provided website or project URL and generates prompt ideas that can be targeted through GEO-optimized content, blogs, and landing pages to improve AI search visibility.
Run Analysis
The Run Analysis section allows users to generate prompt suggestions for a selected project or website.
Project or Website URL
Users can select an existing project or enter a website URL.
The input field supports:
* Selecting a saved project.
* Entering a website URL manually.
* Searching for an existing project.
The selected website is used to analyze the brand and generate relevant ranking prompts.
Find Prompts
After selecting a project or entering a website URL, users can click Find Prompts to start the analysis.
The system analyzes the website and identifies:
* AI search prompts related to the brand.
* High-potential prompt opportunities.
* Prompt difficulty estimates.
* Content ideas for improving Generative Engine Optimization (GEO).
GEO Tip
The top information banner provides guidance on how to use the Prompt Ranking feature effectively.
It explains:
* Enter the website URL to generate ranked prompt ideas.
* Create GEO-optimized landing pages based on the generated prompts.
* Publish blog content targeting those prompts.
* Improve visibility across AI search engines and Large Language Models (LLMs).
The GEO Tip also highlights that optimizing content around these prompts increases the likelihood of AI platforms recommending the brand.
History
The History section displays all previously analyzed websites and projects.
For each analysis, the following information is displayed:
* Website URL
* Website Title
* Date and Time of Analysis
Users can review previous analyses without running the process again.
Export CSV
Users can export the complete Prompt Ranking history as a CSV file for reporting, analysis, or record-keeping.
Refresh
The Refresh option reloads the history list and displays the most recent analyses.
Delete
Users can remove individual analysis records from the history list by clicking Delete.
Deleting a history entry removes only the saved record and does not affect the associated project or website.






Blazly GEO landing page
GEO Landing Page Generator
The GEO Landing Page Generator enables users to create AI-optimized landing pages designed for Generative Engine Optimization (GEO) and Answer Engine Optimization (AEO). By analyzing the target keyword and website, the tool generates a complete landing page with SEO-friendly content, structured sections, and editable HTML that can be published directly to supported CMS platforms.
Create Landing Page
The Create Landing Page section allows users to generate AI-powered landing pages by providing the required information.
Target Keyword
Users must enter the primary keyword they want the landing page to rank for.
The target keyword is used to:
* Define the landing page topic.
* Generate GEO-optimized content.
* Optimize headings and page structure.
* Improve ranking across search engines and AI platforms.
Website URL
Users must provide the website URL.
The system analyzes the website to understand:
* Business niche.
* Brand identity.
* Website content.
* Design preferences.
This information helps generate a landing page aligned with the business.
Content Language
Users can specify the language in which the landing page should be generated.
Supported examples include:
* English
* Spanish
* French
* Dutch
If no language is provided, the landing page is generated in English by default.
Landing Page Generator
Users can choose the AI model used to generate the landing page.
Available options include:
* Gemini (Default)
* Claude (if available)
Different models may provide variations in content quality and writing style.
Create Landing Page
After entering the required information, users can click Create Landing Page.
The system generates:
* GEO & AEO optimized landing page.
* Structured webpage content.
* AI-friendly layout.
* Editable HTML.
* Search-optimized headings and sections.
Design Template
After generation, the system automatically applies a landing page template.
The template is selected based on:
* Website branding.
* Available website styles.
* Layout preferences.
If the website theme cannot be detected, the system applies a default optimized template while maintaining keyword relevance.
Save to Project
Generated landing pages are automatically associated with the selected project.
Users can continue editing the landing page at any time before publishing.
Site URL for Publish (Optional)
Users can optionally provide a publishing URL.
This URL overrides the default project URL when publishing to supported CMS platforms such as:
* WordPress
* Webflow
Generated Landing Page
The Generated Landing Page section displays the complete AI-generated landing page in an editable preview.
Users can review and modify the page before publishing.
Landing Page Content
The landing page contains structured sections optimized for SEO, GEO, and AI search visibility.
Typical sections include:
* Hero Section
* Primary Heading (H1)
* Introduction
* Benefits
* Features
* Call-to-Action (CTA)
* Supporting Content
* Frequently Asked Questions (FAQs)
* Conclusion
Preview
The Preview option displays the landing page in a visual webpage format.
Users can review:
* Layout
* Typography
* Content structure
* Overall appearance
before publishing.
Code
The Code option switches from the visual preview to the HTML source.
Users can:
* View generated HTML.
* Edit page markup.
* Copy raw HTML.
* Integrate with custom websites or CMS platforms.
Regenerate
Users can click Regenerate to create a new version of the landing page.
The system generates fresh content while maintaining the selected keyword and website context.
Copy
Users can click Copy to copy the generated landing page content or HTML to the clipboard.
This allows quick integration into external editors or CMS platforms.
Save
Users can click Save to store the generated landing page within the selected project.
Saved landing pages remain available for future editing and publishing.

The generated landing page follows a structured layout designed for both traditional search engines and AI-powered answer engines.

Blazly Backlinker CSV report
CSV Import
The CSV Import feature allows users to upload a CSV file containing website, blog, or page URLs. Blazly automatically analyzes each URL, discovers available contact email addresses, and prepares personalized outreach emails for backlink campaigns.
This feature is designed to simplify bulk prospecting and streamline the backlink outreach process.
Upload CSV
Users can upload a CSV file containing the URLs they want to analyze.
To upload a file:
1. Click Choose File.
2. Select a valid CSV file from your device.
3. Verify that the correct file has been selected.
Each row in the CSV should contain one website, blog, or page URL.
CSV Format
The uploaded CSV must follow these requirements:
* One URL per row.
* URLs should be valid and publicly accessible.
* The CSV may contain website URLs, blog URLs, or individual page URLs.
For Gmail sending, users should also include consent-related columns in the CSV, such as:
* consented (yes/true)
* consent_source
* consent_date
* consent_notes
If consent information is not included in the CSV, users are responsible for maintaining proof of consent for each lead individually.
Find Emails
After uploading the CSV, click Find Emails to begin processing.
Blazly will:
* Analyze each submitted URL.
* Search for publicly available contact email addresses.
* Identify outreach opportunities.
* Generate personalized outreach emails for the discovered contacts.
The processing time depends on the number of URLs in the uploaded CSV.
Monthly URL Limit
The dashboard displays the number of URLs available for processing during the current billing period.
For example:
* 200 of 200 URLs Left This Month indicates that all monthly URL credits are still available.
* The remaining count decreases as URLs are processed.
CSV Import Workflow
1. Prepare a CSV file with one URL per row.
2. Click Choose File and upload the CSV.
3. Review the selected file.
4. Click Find Emails.
5. Blazly processes each URL and searches for contact email addresses.
6. The discovered contacts can then be used to generate personalized outreach emails and launch backlink campaigns.

Profile Enhancer
The Profile Enhancer helps users optimize their social media profiles using AI. It retrieves profile information from a connected social media account, combines it with the saved website and brand context, and generates suggestions to improve profile quality, branding, and discoverability.
The generated suggestions are for review only and are not published automatically.
Select Platform
Users must choose the social media platform they want to optimize.
Supported platforms include:
* Instagram
* LinkedIn (Upgrade Required)
* LinkedIn Company (Upgrade Required)
* Twitter (X) (Upgrade Required)
* Facebook Page (Not Connected)
* Threads (Not Connected)
Some platforms require a higher subscription plan or an active account connection before they can be used.
Current Profile Text
After selecting a platform, Blazly Social automatically retrieves the current profile information whenever possible.
The editable profile text may include:
* Username
* Bio
* Website
* Other available profile information
If the profile information cannot be retrieved completely, users can manually edit or add the missing details before generating suggestions.
Enhance Profile
Click Enhance Profile to generate AI-powered profile recommendations.
The AI analyzes:
* Existing profile information
* Connected website context
* Brand voice
* Business messaging
It then suggests improvements that make the profile more engaging, professional, and optimized for the selected platform.
LinkedIn PDF Support
For LinkedIn profiles, users can also upload a PDF version of their profile.

AI Recommendations

Blazly Social omnibox
Omnibox
The Omnibox allows users to manage and respond to comments from their connected social media accounts in a single place. It retrieves recent posts from supported platforms, loads their comments, and helps users generate AI-powered replies to improve engagement.
Replies generated by AI are only suggested for comments that have not already received a response.
Supported Integrations
Before using the Omnibox, users must connect at least one supported social media account through the Integrations page.
Supported platforms include:
* LinkedIn
* Twitter (X)
* Facebook Page
* Instagram
* Threads
Only connected platforms will be available for selection.
Select Platform
Choose the social media platform whose posts you want to manage.
Once selected, Blazly Social retrieves the latest posts from the connected account.
Refresh Posts
Click Refresh Posts to synchronize the latest posts from the selected social media platform.
This ensures that recently published posts are available for comment management.
Your Posts
The Your Posts section displays recent posts from the selected platform.
For each post, the following information is displayed:
* Post content preview
* Published date and time
* Number of comments
* Post ID
Select a post to load all of its comments.
Enter Post ID Manually
If a specific post is not listed, users can manually enter the Post ID to load its comments.
This is useful for managing older posts or posts that are not included in the recent post list.
Comments
After selecting a post, the Comments section displays all comments associated with that post.
Users can:
* Read customer comments.
* Review existing replies.
* Respond to comments manually.
* Generate AI-powered replies for eligible comments.
Comments that already have replies from the connected social media account are identified and excluded from automatic AI responses.
Enable AI
The Enable AI toggle activates automatic AI reply generation for comments that have not yet been answered.
When enabled:
* AI generates suggested replies using Google Gemini.
* Only unanswered comments are processed.
* Existing replies from the connected platform are preserved.
* Users can review AI-generated responses before posting.


Blazly social your posts
Your Posts
The Your Posts page displays all posts created or scheduled within Blazly Social. It serves as a centralized location where users can view and manage posts generated through Create Post, Social Autopilot, and the Calendar.
Users can review post details, monitor publishing status, and filter posts by social media platform.
Social App Filter
The Social App dropdown allows users to filter posts by connected social media platform.
Available options include:
* All Platforms
* Instagram
* Facebook
* LinkedIn
* Twitter (X)
* Threads
Selecting a platform displays only the posts associated with that social media account.
Refresh
Click Refresh to load the latest posts and update the page with any newly created, scheduled, or published content.

Each post includes a status that indicates its current publishing stage.
Common statuses include:
* Draft – The post has been created but has not been published.
* Scheduled – The post is scheduled to be published at a specified date and time.
* Published – The post has been successfully published to the selected platform.
Scheduled Date and Time
Each scheduled post displays its planned publishing date and time, allowing users to monitor upcoming content.
Post Content
The post preview displays the content that will be or has been published.
This may include:
* Caption or text
* Emojis
* Hashtags
* Mentions
* Formatting
Users can review the content before publication.
Media Preview
If a post contains media, a preview is displayed alongside the post.
Supported media types include:
* Images
* Graphics
* Promotional banners
* Other uploaded visual content
This allows users to verify that the correct media is attached to the post.
Draft Management
Draft posts can be reopened for further editing.
Click Open Draft to:
* Modify the post content.
* Update attached media.
* Change the scheduled publishing date and time.
* Save the updated draft or schedule it for publishing.
Autopilot Drafts
The page also displays the number of posts generated by Social Autopilot that are currently saved as drafts.
These drafts can be reviewed, edited, approved, or scheduled before they are published.
